<context>
# Root cause from UAT
- WebSocket closes immediately after connecting
- Error: "WebSocket is already in CLOSING or CLOSED state"
- Audio is captured and sent, but connection dies before API can process

# Working config (geminiLive.ts lines 127-138)
```
config: {
  responseModalities: [Modality.AUDIO],
  speechConfig: {
    voiceConfig: { prebuiltVoiceConfig: { voiceName } },
  },
  systemInstruction: config.systemInstruction!,
  outputAudioTranscription: {},
  inputAudioTranscription: config.inputAudioTranscription || {},
}
```

# Broken config (useLiveAPISpeechRecognition.ts lines 171-175)
```
config: {
  responseModalities: [Modality.TEXT],  // <-- WRONG!
  inputAudioTranscription: {},
  systemInstruction: systemInstruction || 'Transcribe user speech only. No response needed.',
}
```

# Key differences causing the issue
1. responseModalities: TEXT vs AUDIO - TEXT expects model to generate text response, causing immediate turnComplete
2. Missing speechConfig - required for proper session handling
3. Missing outputAudioTranscription - should be present
4. System instruction causing model to try responding

# Solution
Change config to match working voice pattern:
- Use Modality.AUDIO (like working config)
- Add speechConfig (even if empty voice config)
- Add outputAudioTranscription: {}
- Keep inputAudioTranscription: {}
- Simplify system instruction to not trigger response
</context>
