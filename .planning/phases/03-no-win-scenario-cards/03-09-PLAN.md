<objective>
Remove green/red moral framing from swipe UI and randomize left/right choice assignment to align with the no-win (Kobayashi Maru) simulation design.

Problem: Green = right/good, Red = left/bad creates a psychological frame that undermines the no-win thesis. Players perceive one side as "correct" when both choices should feel equally consequential.

Solution:
1. Neutralize all swipe-related colors (preview labels, mobile hints, CSS gradients)
2. Use amber/cyan status framing in FeedbackOverlay (cost vs proceed, not good vs bad)
3. Randomly swap onLeft/onRight at deck shuffle time so choices aren't consistently on one side

Output: Updated UI with neutral framing + randomized choice placement + unit tests.
</objective>

<context>
# Architectural Decision: Deck-Level Swap

The randomization of left/right choice assignment happens in `shuffleDeck()` in `lib/deck.ts`.
This means card data is swapped once at game start — all downstream code (CardStack, FeedbackOverlay,
useGameState, history tracking) works unchanged because `onLeft` is always what's on the left.

Alternative considered: presentation-level swap with `isFlipped` boolean per card.
Rejected because every consumer of `onLeft`/`onRight` would need flipping awareness — high coupling risk.

# Color Design Decision: Amber/Cyan Status Frame

- Amber (fine > 0): signals "this had a cost" — factual, not punitive. Matches existing amber usage for high-heat warnings.
- Cyan (fine === 0): signals "acknowledged, proceed" — neutral, matches the game's accent color.
- This replaces green (good) / red (bad) moral framing with a cost/no-cost status frame.

# CSS Gradient Decision: Neutralize to Amber/Cyan

The `.swipe-gradient-left` and `.swipe-gradient-right` CSS classes are currently unused in CardStack
but retained and updated to amber/cyan in case they're referenced in the future.
</context>

    Modify `shuffleDeck()` to randomly swap `onLeft` and `onRight` for each card during the shuffle.

    After the Fisher-Yates shuffle loop, iterate through the shuffled array. For each card,
    generate `Math.random() < 0.5` — if true, create a new card object with `onLeft` and `onRight` swapped.

    Implementation:
    ```typescript
    // After Fisher-Yates shuffle, randomly swap left/right choices per card
    for (let i = 0; i < shuffled.length; i++) {
      if (Math.random() < 0.5) {
        const card = shuffled[i];
        shuffled[i] = { ...card, onLeft: card.onRight, onRight: card.onLeft };
      }
    }
    ```

    This ensures:
    - ~50% of cards have swapped sides
    - Each game session has a different layout
    - All downstream code works unchanged (onLeft is always what appears on the left)
  </action>
