## Key Design Patterns (from RESEARCH.md)

### Incident Categories (6 total) — Design 2+ base scenarios per category, then role-specific variants:

1. **No-Win Dilemmas (6+ cards)** — Generic governance/tradeoff dilemmas
   - Example: "Speed vs. Security" (push to production vs. delay for audit)
   - Both outcomes carry budget + heat + reputation impact

2. **Prompt Injection (2+ cards)** — Security vulnerability in AI feature
   - Real 2024-2025 incidents: GitHub Copilot RCE (CVE-2025-53773), Cursor IDE RCE (CVE-2025-54135/54136), financial services injection attacks, Slack AI exfiltration
   - Dilemma: Fix in production (immediate risk) vs. pause feature (revenue loss)
   - Role variants: CEO sees liability; Engineer sees technical fix; Data Scientist sees model impact

3. **Model Drift / Retraining (2+ cards)** — AI accuracy degradation
   - Real 2024-2025 incidents: 75% of businesses saw performance decline, 9.3% improvement from adaptive retraining
   - Dilemma: Retrain (expensive, resource burn) vs. freeze (accuracy loss, customer complaints)
   - Role variants: Finance sees cost; Engineer sees timeline; Scientist sees technical debt

4. **Explainability / Black Box (2+ cards)** — Audit pressure vs. deployment speed
   - Real 2024-2025 incidents: Black-box-only audits insufficient for compliance, regulators identify explainability as top concern
   - Dilemma: Deploy fast (keep black box) vs. spend time explaining (miss deadline, satisfy audit)
   - Role variants: CEO sees audit risk; Dev sees timeline; Auditor sees compliance gap

5. **Shadow AI / Unauthorized Tools (2+ cards)** — Governance vs. innovation
   - Real 2024-2025 incidents: 78% employee adoption, 60.2% use tools, 13% report harm from shadow AI
   - Dilemma: Allow tool use (stifle innovation) vs. enforce governance (data leakage, compliance risk)
   - Role variants: CEO sees liability; Engineer sees job security; InfoSec sees exposure

6. **Copyright / Data Provenance (2+ cards)** — Cost vs. legal exposure
   - Real 2024-2025 incidents: 70+ lawsuits by 2025, EU AI Act requires transparency
   - Dilemma: Use cheap data (unknown provenance) vs. pay for clean data (budget drain)
   - Role variants: Finance sees cost; Legal sees liability; ML sees model quality

### Reusability Patterns (CRITICAL for 80+ cards):

**Pattern A: Same incident, different context**
- Example: "Prompt injection in production"
  - Variant 1: "Security team discovers during pen test" (internal discovery, time to fix)
  - Variant 2: "Customer discovers via API call" (public disclosure, reputation loss)
  - Both are separate cards, can target different roles

**Pattern B: Same incident, different outcome focus**
- Example: "Model drift detected"
  - **CEO version:** "Accuracy dropped 15%. Fix costs $500K. What do you do?"
    - Left: "Retrain immediately" (budget burn, delay launch)
    - Right: "Freeze predictions" (revenue loss, alarm customers)
  - **Engineer version:** "You spotted model accuracy drop. What's your move?"
    - Left: "Retrain on new data" (adds 3 weeks, technical debt)
    - Right: "Patch feature pipeline" (risky, might mask real problem)
  - **Scientist version:** "New training data arrived. Model degrading. Investigate or deploy?"
    - Left: "Investigate drift" (lose 2 weeks, clients expect updates)
    - Right: "Deploy anyway" (risk wrong decisions, audit exposure)

### Card Structure (from RESEARCH.md Code Examples)

```typescript
{
  id: "NOWIN-[TYPE]-[ROLE]-[NUM]",    // E.g., "NOWIN-INJECTION-DEV-01"
  source: AppSource,                   // SLACK, EMAIL, TERMINAL, IDE, JIRA, NOTION, MEETING
  sender: "who_raised_it",             // Role or system (e.g., "SECURITY_TEAM", "CUSTOMER")
  context: "incident_type",            // E.g., "SECURITY", "COMPLIANCE", "BUDGET"
  storyContext: "scene_setting",       // 1-2 sentence immersion (pressure, timeline, stakes)
  text: "the_dilemma",                 // The incident + request (framed as choice)
  onRight: {
    label: "option_text",              // E.g., "Deploy", "Ignore", "Escalate"
    hype: +number or -number,          // Market perception change
    heat: +number or -number,          // Regulatory/legal heat
    fine: large_number,                // Potential fine (0 if compliant)
    violation: "statute_or_None",      // Legal consequence
    lesson: "explain_tradeoff",        // NOT "right answer", explain the cost
    feedback: {
      ROASTER: "cynical_takedown",
      ZEN_MASTER: "philosophical_wisdom",
      LOVEBOMBER: "enthusiastic_spin"
    }
  },
  onLeft: { /* mirror structure, different outcome */ }
}
```

**Critical constraint:** Both onLeft and onRight must show penalties. If one path is "free" (zero heat, zero fine), it's a logic puzzle (violates no-win pattern).

## Personality Tone Guide

Use existing 3-voice system (already proven in Phase 01):

- **ROASTER (PersonalityType.ROASTER):** Cynical, sarcastic, judgmental
  - Example: "That's called Fraud, darling. The SEC doesn't have a sense of humor."
  - Tone: Mocks the player's choice, points out obvious consequences

- **ZEN_MASTER (PersonalityType.ZEN_MASTER):** Philosophical, reflective, wisdom
  - Example: "Manifesting wealth from nothing... also manifesting prison time."
  - Tone: Calm acknowledgment of tradeoff, suggests deeper lesson

- **LOVEBOMBER (PersonalityType.LOVEBOMBER):** Enthusiastic, supportive, excited
  - Example: "Manifestation is real!! The investors will literally love the vibe!"
  - Tone: Celebrates the choice, spins positive but ignores consequences

**Key:** Voices must reference the *specific outcome* (what was sacrificed, what was gained), not generic praise/blame.

## Common Pitfalls to Avoid (from RESEARCH.md)

1. **Dominant Strategy:** One path is clearly better. Solution: Ensure both have comparable penalties.
2. **Time Pressure on All Cards:** Fatigue over impact. Solution: Reserve countdown for ~20% (high-stakes decisions).
3. **Lesson Declares Winner:** "Best practice: always X." Solution: Frame as "Why both paths cost differently."
4. **Copy-Paste Across Roles:** Same text for all roles. Solution: Explicitly reframe for each role's unique risk exposure.
5. **Debrief Doesn't Connect:** Lesson doesn't reference player's specific choice. Solution: Lesson should name the outcome and its cost.
6. **Incident Fictionality:** Made-up scenarios feel hollow. Solution: Source all from 2024-2025 documented cases.
