## Open Issues from ISSUE-LOG.md

### Issue 3: BGM autoplay doesn't start on swipe — UNRESOLVED
**Root cause:** iOS Safari autoplay policy — swipe gesture touchend is NOT trusted for `el.play()`

**Fix attempts tried:**
- touchstart-based unlock — rejected by iOS 9+
- touchend-based unlock — works for taps, fails for swipes  
- Play/pause primer tricks — pause revokes trusted-media status
- All-events unlock (current) — tap works, swipe fails

**Platform constraint:** iOS Safari only grants autoplay trust from `click` and short-tap `touchend`. Swipe gestures never satisfy policy.

**Paths forward (NOT YET TRIED):**
- **Option A — BGM via AudioBufferSourceNode:** Fetch/decode music, play via buffer source. Downside: ~10-30MB memory per track.
- **Option B — Early unlock via pre-game tap:** Boot system → personality → role buttons should unlock. Verify if `el.play()` is actually failing or succeeding silently.
- **Option C — "Tap to start audio" overlay:** Industry-standard iOS pattern, show once on iOS mobile.

**Decision needed:** Which option to pursue? BGM not starting on first swipe is acceptable UX or needs fix?

### Issue: Flying speed slower on mobile — PARTIALLY FIXED
**Root cause 1 (FIXED):** No delta time — 120Hz vs 60Hz difference. Fixed via delta normalization.

**Root cause 2 (NOT FIXED):** Fixed x∈[-800,800], y∈[-600,600] initialization. On mobile (~375px canvas), only center stars visible → slow lateral motion.

**Fix needed:** Initialize x/y relative to screen size in StarfieldBackground.tsx:
```typescript
reset(halfW: number, halfH: number) {
  const xRange = (halfW * Z_MAX) / FOCAL_LENGTH;
  const yRange = (halfH * Z_MAX) / FOCAL_LENGTH;
  this.x = Math.random() * xRange * 2 - xRange;
  this.y = Math.random() * yRange * 2 - yRange;
}
```

**Status:** Implementation in progress in uncommitted StarfieldBackground.tsx changes.

### Issue 2: Voice muted when BGM paused — HAS WORKAROUND
**Current fix:** Never call `el.pause()`. Set `gainRef.current.gain.value = 0` instead. MediaElement plays silently; iOS audio session uninterrupted; voice stays running.

**Complexity:** All volume effects must guard `if (!enabledRef.current) return` to prevent overriding the 0 gain.

**Status:** Working but complex. Simpler solution may exist.
