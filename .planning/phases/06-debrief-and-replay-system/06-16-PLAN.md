The current implementation uses `window.open(linkedInUrl, "_blank", "width=550,height=420")` which can be blocked by browser popup blockers. The improved approach uses LinkedIn's share-offsite URL format:

```
https://www.linkedin.com/sharing/share-offsite/?url={gameUrl}&summary={preFilledText}
```

This opens LinkedIn's native share dialog where users can review and edit the pre-filled content before posting.

Key insight from research: LinkedIn's share-offsite URL supports a `summary` parameter that pre-fills the share text. This is the recommended approach for third-party sharing without OAuth.

<task type="auto">
  <name>Task 2: Refactor DebriefPage3Verdict to use anchor tag</name>
  <files>components/game/debrief/DebriefPage3Verdict.tsx</files>
  <action>
Replace the button with window.open() handler with an anchor (<a>) tag that uses the LinkedIn share URL as href. This avoids popup blockers and is more reliable.

Current implementation:
```tsx
const handleShareLinkedIn = () => {
  if (!role || !archetype) return;
  const linkedInUrl = getShareUrl(role, archetype, resilienceScore);
  window.open(linkedInUrl, "_blank", "width=550,height=420");
};

<button onClick={handleShareLinkedIn}>
  <i className="fa-brands fa-linkedin"></i>
  Share to LinkedIn
</button>
```

New implementation:
```tsx
const linkedInShareUrl = role && archetype 
  ? getShareUrl(role, archetype, resilienceScore)
  : null;

<a 
  href={linkedInShareUrl || "#"}
  target="_blank"
  rel="noopener noreferrer"
  className="...same button styling..."
  aria-label="Share to LinkedIn"
>
  <i className="fa-brands fa-linkedin"></i>
  Share to LinkedIn
</a>
```

Key changes:
1. Compute share URL in render (not on click)
2. Use anchor tag with target="_blank"
3. Add rel="noopener noreferrer" for security
4. Preserve all existing styling (Tailwind classes)
5. Handle null case gracefully (href="#" when data missing)

This approach:
- Works with browser "open in new tab" behavior
- Avoids popup blocker issues
- Is accessible (proper anchor semantics)
- Can be tested by checking href attribute
